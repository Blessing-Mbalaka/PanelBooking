# Render Deployment Guide

## Quick Setup (5 minutes)

### 1. Prepare Environment Variables

Create a `.env` file (never commit this):
```
SECRET_KEY=your-very-long-secret-key-here
DEBUG=False
ALLOWED_HOSTS=your-app.onrender.com,localhost
```

Generate a strong SECRET_KEY:
```bash
python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"
```

### 2. Push to GitHub

```bash
git init
git add .
git commit -m "Deploy to Render"
git push origin main
```

### 3. Create Render Web Service

1. Go to [render.com](https://render.com) and sign in
2. Click **+ New** â†’ **Web Service**
3. Connect your GitHub repo
4. Fill in:
   - **Name**: `booking-platform`
   - **Environment**: `Python 3.12`
   - **Region**: Choose closest to you
   - **Build Command**: `pip install -r requirements.txt && python manage.py migrate && python manage.py collectstatic --noinput`
   - **Start Command**: `gunicorn booking_platform_backend.wsgi:application --bind 0.0.0.0:$PORT`

5. Click **Advanced** â†’ **Add Environment Variable**:
   - `SECRET_KEY`: Paste your generated key
   - `DEBUG`: `False`
   - `ALLOWED_HOSTS`: `your-app.onrender.com,localhost`

6. Click **Create Web Service** and wait ~5-10 minutes for deployment

### 4. Initialize Database & Admin

After deployment, open the Render dashboard and run in the **Shell**:

```bash
python manage.py migrate
python manage.py createsuperuser
python manage.py load_supervisors extract  # If you've already uploaded supervisor mappings
```

Or go to `/admin/` to manually add superuser via web interface.

### 5. Test Live

- Visit: `https://your-app.onrender.com/`
- Admin panel: `https://your-app.onrender.com/admin/`
- Export bookings: `https://your-app.onrender.com/exportdata/`

| File | Purpose |
|------|---------|
| `requirements.txt` | Python dependencies (updated with gunicorn, whitenoise) |
| `.env.example` | Template for environment variables |

| `Procfile` | Process definitions for Render |
| `render.yaml` | Alternative Render config (optional) |
| `static/` | Collected static files (JS, HTML) |
| `staticfiles/` | Generated by `collectstatic` |

## Troubleshooting

**502 Bad Gateway**
- Check Render logs: Dashboard â†’ Your App â†’ Logs
- Verify `gunicorn` start command
- Ensure `DEBUG=False` not causing template errors

**Static files not loading**
- Run: `python manage.py collectstatic --noinput`
- Verify `STATIC_ROOT` and `STATICFILES_DIRS` in settings
- Check whitenoise is in MIDDLEWARE

**Admin page 404**
- Ensure migrations ran: `python manage.py migrate`
- Check `ALLOWED_HOSTS` includes your domain

**Database errors**
- Render provides free SQLite (persists across deploys)
- For production scaling, upgrade to PostgreSQL via Render dashboard

## Scaling Up (Optional)

When ready for production data:

1. **Upgrade Database** (Render Dashboard):
   - Render â†’ Your App â†’ Environment â†’ Add PostgreSQL
   - Set `DATABASE_URL` env var

2. **Update `settings.py`** to use PostgreSQL:
   ```python
   import dj_database_url
   DATABASES = {
       'default': dj_database_url.config(default=os.getenv('DATABASE_URL'))
   }
   ```

3. Add `dj-database-url` to `requirements.txt`

4. Redeploy (Render auto-migrates)

## API Endpoints

- `GET /` - Main booking page
- `GET /api/schedule/` - Schedule config
- `GET /api/bookings/` - List bookings
- `POST /api/bookings/` - Create booking
- `POST /api/bookings/<id>/cancel/` - Cancel booking
- `GET /api/system-counts/` - Student/supervisor counts
- `GET /api/supervisors/search/?q=name` - Search supervisors
- `GET /exportdata/` - Download bookings as CSV
- `GET /admin/` - Django admin panel
